# Daily Research Checklist

퀵실버는 매일 포스트 작성 전에 아래 순서로 조사한다.

## 0. 포스트 기준 시각
- 포스트 제목/날짜는 실제 작성 시각(Asia/Seoul)으로 입력
- 미래 날짜 금지
- 파일명도 실제 작성일 기준

## 1. 공식 소스 우선 확인

### OpenAI
- https://openai.com/news/
- https://openai.com/index/
- 새 모델 / Safety / API / 제품 발표 확인

### Anthropic
- https://www.anthropic.com/news
- Claude / API / 정책 / 보안 관련 발표 확인

### Google / Gemini
- https://blog.google/technology/ai/
- https://developers.googleblog.com/
- https://ai.google.dev/
- Gemini / AI Studio / API / 개발자 공지 확인

### xAI / Grok
- xAI 공식 블로그/공식 채널 확인
- 직접 접근이 막히면 신뢰 가능한 보조 소스로 교차 확인

### 중국 모델/기업
- Qwen
- DeepSeek
- Zhipu
- Moonshot
- MiniMax
- Baidu
- Tencent
- ByteDance 계열
- 공식 블로그, GitHub, 발표문, 모델 릴리스 중심 확인

## 2. 커뮤니티 / 반응 체크

### Hacker News
- 메인 피드에서 AI 관련 항목 확인
- 관련 item 링크까지 확보

### Reddit
- r/LocalLLaMA
- r/MachineLearning
- r/OpenAI
- r/singularity
- 추출 제한이 있으면 제목/링크 중심으로 참고하고 과장된 반응은 걸러낼 것

### GitHub
- Trending
- 주요 모델/에이전트/툴 릴리스
- 급상승 저장소 확인

### YouTube
- 주요 공식 채널 발표 영상
- 데모/키노트/연구 발표 확인
- 필요 시 summarize 계열 도구로 본문/요약 확보

## 3. 분석 포인트 정리
각 뉴스마다 최소한 아래를 메모:
- 무슨 일인가
- 왜 중요한가
- 개발자에게 어떤 영향이 있나
- 시장/제품 경쟁 관점에서 무슨 의미인가
- 공식 발표와 실제 의미 사이에 온도차가 있나

## 4. 출처 규칙
- 공식 원문 링크 우선
- 가능하면 보조 출처 1개 이상 확보
- 수치/가격/벤치마크는 원문 확인 전 단정 금지
- 루머는 루머라고 명시

## 5. 게시 품질 기준
다음 중 2개 이상 충족 못 하면 더 조사 후 게시:
- 오늘의 큰 흐름이 명확함
- 핵심 뉴스 3개 이상이 충분히 설명됨
- 출처가 충분함
- 개발자 관점 정리가 있음
- 커뮤니티 반응이 단순 링크 나열이 아님

## 6. 기본 운영 원칙
- 적게 써도 되니 부실하게 쓰지 말 것
- 억지로 모든 회사를 다 넣지 말 것
- 중요한 날은 심층 분석 1개 + 브리핑 1개로 분리 가능
- 발표문을 그대로 번역하지 말고 의미를 해석할 것
