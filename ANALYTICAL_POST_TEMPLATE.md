# Analytical AI Briefing Template

매일 AI 브리핑 포스트는 아래 구조를 기본으로 사용한다.

```md
---
layout: post
title: "[AI 브리핑] YYYY-MM-DD HH:mm KST 오늘의 AI 뉴스"
date: YYYY-MM-DD HH:mm:00 +0900
categories: [ai, briefing]
tags: [ai, openai, anthropic, gemini, grok, qwen, deepseek, hn, reddit]
---

## 한눈에 보기
- 오늘 가장 중요한 변화 3~5개
- 단순 사실이 아니라 흐름 중심으로 요약

## 오늘의 핵심 흐름
- 오늘 AI 업계에서 가장 중요한 큰 방향 1~3개
- 예: 소형 모델 경쟁, 에이전트 보안, 가격 인하, 오픈소스 가속 등

## 핵심 뉴스 분석
### 1. [뉴스 제목]
**무슨 일이 있었나**
- 발표/출시/업데이트의 핵심 사실

**왜 중요한가**
- 이 변화가 업계에서 가지는 의미

**누가 영향을 받나**
- 개발자
- 스타트업/비즈니스
- 일반 사용자

**퀵실버 해석**
- 홍보 문구를 걷어낸 현실적 해석
- 이전 흐름과 연결되는지 여부
- 과대평가/과소평가 포인트

**출처**
- [공식 원문](https://example.com)
- 필요 시 [보조 출처](https://example.com)

### 2. [뉴스 제목]
(동일 구조 반복)

## 개발자 관점 체크포인트
- API / SDK / 가격 / 레이트리밋 / 모델 접근성 변화
- 실제 써볼 가치가 있는지
- 기존 워크플로를 바꿀 만한지

## 시장/업계 관점
- 어느 회사가 어떤 방향으로 가는지
- 경쟁 구도가 어떻게 바뀌는지
- 단기 이슈인지 장기 추세인지

## 커뮤니티 반응
### Hacker News
- 실제 반응 요약
- 의미 있는 링크 남기기

### Reddit
- 과열된 반응과 실사용자 반응 구분

### GitHub / YouTube / 기타
- 릴리스, 데모, 주요 영상, 오픈소스 동향

## 오늘의 결론
- 오늘 뉴스들을 한 문단으로 관통하는 해석

## 참고 출처 모음
- 링크를 한곳에 다시 정리
```

## Writing Rules

- 기사체 요약보다 분석형 문장 우선
- 핵심 뉴스는 보통 3~6개 정도가 적당
- 출처는 가능한 한 공식 원문 우선
- 출처는 클릭 가능한 마크다운 링크로 작성
- 공식 발표와 커뮤니티 해석을 반드시 구분
- 확인이 어려운 정보는 억지로 넣지 말 것
- 추출이 부실하거나 확인이 안 되는 소스는 과감히 제외할 것
- 별로 중요하지 않은 뉴스를 개수 채우려고 넣지 말 것
- 핵심 포인트는 **볼드**를 적절히 사용해 가독성을 높일 것
